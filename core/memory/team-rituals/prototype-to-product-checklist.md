## Evidencia de problema

- [ ] Persona ou ICP definido.
- [ ] Dor descrita com contexto real.
- [ ] Evidencia inicial coletada com usuario, cliente, mercado ou operacao.
- [ ] Metrica de sucesso inicial definida.
- [ ] Alternativas existentes mapeadas.

## Prototipo

- [ ] README com como rodar, abrir ou entender a demo.
- [ ] Demo funcional, navegavel ou demonstravel.
- [ ] Fluxo principal claro.
- [ ] Limitacoes conhecidas documentadas.
- [ ] Criterios de aceite definidos.
- [ ] Proximos passos registrados.

## Validacao

- [ ] Feedback registrado.
- [ ] Hipoteses principais listadas.
- [ ] Resultado da validacao documentado.
- [ ] Evidencias anexadas.
- [ ] Aprendizados traduzidos em decisoes.

## Produto e negocio

- [ ] Proposta de valor clara.
- [ ] Caso de uso com potencial de venda, upsell, pacote de servico ou acelerador interno.
- [ ] Hipotese de embalagem comercial definida.
- [ ] Hipotese de precificacao ou modelo de oferta definida.
- [ ] Canal de venda ou cliente piloto sugerido.
- [ ] Concorrentes/substitutos considerados.

## Tecnologia

- [ ] Viabilidade tecnica revisada por TL/arquitetura.
- [ ] Dependencias principais conhecidas.
- [ ] Custos tecnicos iniciais estimados.
- [ ] Plano de MVP com escopo minimo.
- [ ] Integracoes e dados mapeados.
- [ ] Reuso do Feltrim's Framework avaliado.

## QA, seguranca e operacao

- [ ] Plano minimo de QA definido.
- [ ] Riscos de seguranca e privacidade avaliados.
- [ ] Nenhuma credencial ou dado sensivel exposto.
- [ ] Uso de dados de cliente autorizado, quando aplicavel.
- [ ] Riscos de suporte e operacao mapeados.
- [ ] Observabilidade ou evidencias tecnicas planejadas, quando necessario.

## Go/No-Go

- [ ] Marlon-PO validou problema, valor e hipotese comercial.
- [ ] Beatriz-TL validou viabilidade tecnica.

- [ ] Lideranca humana aprovou continuidade, premio real ou uso comercial, quando aplicavel.

## Decisao

Selecione uma opcao:

- [ ] Arquivar: aprendizado registrado, sem continuidade agora.
- [ ] Incubar: precisa de mais validacao ou refinamento curto.
- [ ] MVP: pronto para backlog com escopo minimo.
- [ ] Produto vendavel candidato: pronto para discovery comercial e Go/No-Go executivo.
- [ ] Bloqueado por risco: nao continuar ate resolver risco critico.
